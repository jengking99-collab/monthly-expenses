채팅에서 기본 구성을 해야 코드에서 리소스(컨텍스트)를 절약할 수 있음.

1. 채팅에서 기본 코딩 작업
2. 코딩 작업 후 로컬 시행을 위해 작업 폴더 생성 
3. 채팅에서 app.jsx 파일 생성
4. 채팅에서 코딩 작업 시 새로운 채팅창을 만들면 컨텍스트는 리셋 됨.
- 단, 플랜 사용은 누적 됨.
5. 폴더에서 다음 명령어를 실행하여 해당 폴더에 'React 프로젝트' 생성
- npm create vite@latest . -- --template react
6. 빌더를 위한 패키지 설치
- npm install
- npm install recharts xlsx
7. scr 폴더에 다운받은 App.jsx 파일로 교체(기존은 기본 소스만 있는 파일 임)
8. Vite 서버 실행
- npm run dev
9. 서버 안내 주소로 접속하여 테스트
- 예 ; http://localhost:5173/
10. 서버 종료
- Ctrl + C
11. 위 작업이 어느정도 마무리 되면 code에서 App.jsx 파일을 포함 시키고 코딩 작업(수정, exe 파일 빌더) 수행
12. code 종료는 exit 명령어 입력
- exit
13. 독립형 exe 빌더 시 프로젝트 폴더에 Electron 세팅 진행

현재 버전에 빌드 에러시 현재버전 다음에 영문을 추가하여 빌드하고, 신규 버전 빌드는 기존과 동일하게 버전 폴더를 생성해서 빌드해줘.

구현된 기능 목록:
① 월 선택 (사이드바)
좌측 사이드바에서 1~12월 선택 가능. 실행 시 오늘 날짜 기준 해당 월 자동 선택.
② 일별 예상 잔액 테이블
날짜별로 수입/지출 항목, 국민은행·신한은행 예상 잔액, 합계 잔액 표시. 오늘 날짜는 파란색으로 강조.
③ 잔액 설정
상단 "잔액 설정" 버튼으로 오늘 기준 국민/신한 실제 잔액 입력 → 이후 날짜 자동 계산.
④ 고정 항목 관리
"고정 항목" 탭에서 일자·항목명·자산·금액·구분 직접 수정. 행 추가/삭제 가능.
⑤ 수동 항목 추가/삭제
매일 행의 "＋ 추가" 버튼 또는 상단 "항목 추가" 버튼으로 해당 날짜에 항목 추가. 태그의 ✕로 삭제.
⑥ 엑셀 내보내기/가져오기
고정항목·잔액설정·수동항목 3개 시트로 구성된 xlsx 저장 및 불러오기.
⑦ 통계 보기
이번 달 / 올해 전체 지출 도넛 차트, 월별 추이 라인 차트, 항목별 비율 상세 목록.

다음 사항을 반영해서 V1.0으로 작성해줘.
항목별 내용으로 적용하되, 단계별 적용 플랜 또는 안정성을 고려한 적용 단계를 구성해서 적용해줘.
또한, 기존 수정과 처음부터 코딩에 대한 판단도 해줘.(코딩 소요시간과 안정성을 고려)
처음부터 적용할 경우 지금까지 작성한 내용중 아래 내용을 기준으로 '기초작업'으로 내용 추가해줘.


-------------- 프로젝트 목적 --------------
1. 월 단위로 고정 또는 변칙적 수입/지출에 대한 관리 위함.
2. 매월 지출되는 금액을 실시간 모니터링하여 오늘 지출 금액에 대한 통장 잔액을 관리.
3. 또한, 월급 이전에 월 지출에 대한 통장별 잔액을 관리하기 위함.


-------------- 프로그램 시작 --------------
*1. 고정지출 저장 데이터가 있으면, 업로드하고 데이터 적용. 없을 경우 빈 데이터로 적용.
*2. 월 지출 저장 데이터가 있으면 업로드하고 적용, 없을 경우 고정지출 데이터를 반영.


-------------- 고정지출 적용 --------------
1. 엑셀파일 업로드는 월별 지출 내용이 아니고, 월 고정지출 항목에 적용.
*2. 고정지출 데이터는 수동저장 버튼을 누르면 자동 내부에 저장되어 다음 프로그램 시작 시 자동 업로드
3. 매월 데이터는 월 고정지출에서 읽어와 업데이트 됨.
*4. 고정지출 내용이 업데이트 되는 시기는 오늘기준 해당 달부터 이후 모두에 적용 됨.(이전 달까지는 적용되지 않음)
5. 고정지출은 엑셀파일로 저장하는 기능도 포함.


-------------- 데이터 저장/관리 --------------
1. 월 지출 모든 데이터는 자동으로 저장하여 다음 프로그램 실행 시 자동으로 업로드 될수 있게 반영.


-------------- 월 지출 --------------
*1. 매월 1일에 이월 금액은 이전달 잔액으로 반영.(매월 1일에는 이전달 최종 잔액을 이번달 이월로 적용)
2. 해당 월 지출 내용을 수동으로 수정하면, 수동저장 버튼을 눌러 저장되어 다음 프로그램 시작 시 업로드되어 반영


-------------- 통계 --------------
1. 통계 데이터는 년 단위로 관리될 수 있도록 통계 부분에 '월'과 '년'을 선택 할 수 있도록 구성.
2. 년도는 2020년부터 2040년까지 선택할 수 있도록 구성
