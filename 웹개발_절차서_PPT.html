Section 1 · 서비스 개요
3가지 서비스 역할 개요
무료 플랜만으로 완전한 웹 앱 운영 인프라를 구성합니다
🐙
GitHub
소스 코드의 유일한 원본(Single Source of Truth). 모든 변경 이력을 추적하고 Vercel 자동 배포를 트리거합니다.
  • 코드 버전 관리 (git)
  • 브랜치 전략 (master / feature/*)
  • Vercel Webhook 자동 호출
  • 코드 백업 & 협업 플랫폼
Free 플랜 · 무료
▲
Vercel
GitHub 코드를 자동 빌드하여 전 세계 CDN에 배포. push 한 번으로 60~90초 안에 전 세계에 반영됩니다.
  • 자동 CI/CD 파이프라인
  • 글로벌 CDN 호스팅
  • 환경변수 안전 관리
  • 즉각 롤백 & 미리보기 URL
Hobby 플랜 · 무료
🔥
Firebase
서버 없이 기기 간 데이터를 동기화합니다. 모바일에서 입력하면 PC에서 자동으로 반영됩니다.
  • Firestore NoSQL 데이터베이스
  • 실시간 기기 간 동기화
  • syncKey 기반 접근 제어
  • 오프라인 장애 시 로컬 동작
Spark 플랜 · 무료
Section 1 · GitHub
🐙 GitHub — 코드 버전 관리
핵심 역할
  • 코드 저장 — 원격 서버에 소스코드 안전 보관
    모든 커밋 이력 영구 보존
  • 버전 관리 — 언제든 과거 버전으로 복귀
    git revert / git checkout
  • 배포 트리거 — master push → Vercel Webhook
    코드 변경 즉시 자동 배포 시작
이 프로젝트 저장소
github.com/jengking99-collab/monthly-expenses
브랜치 전략
master  ──●──────●──────●──  (운영, 자동 배포)
              \            /
feature/xxx  ●──●──●────      (개발, preview URL)
주요 명령어
# 변경사항 저장
git add src/App.jsx
git commit -m "feat(v2.4): 기능명"

# 운영 배포 (Vercel 트리거)
git push origin master

Section 1 · Firebase
🔥 Firebase — 기기 간 데이터 동기화
Firestore 데이터 구조
Firestore
└── sync/
    └── {syncKey}          ← 사용자 비밀 키
        ├── data: string   ← 월별 지출 데이터
        ├── fixed_base: string ← 고정항목 이력
        └── updatedAt: timestamp
무료 한도 (Spark 플랜)
항목무료 한도실사용
읽기50,000건/일~100건/일
쓰기20,000건/일~50건/일
저장1 GiB~1 MB
동기화 방향
  • 앱 → Firestore (자동 push)
    데이터 변경 후 5초 debounce → 자동 전송
  • Firestore → 앱 (자동 pull)
    앱 시작 시 silent pull — 항상 최신 상태
  • 수동 동기화
    사이드바 동기화 버튼 → 즉시 pull
⚠️ 보안 원칙
syncKey가 유일한 접근 제어 수단입니다.
타인과 공유하지 마세요.

핵심 포인트: 개발자는 코드만 push합니다. 나머지(빌드·배포·CDN 전파)는 모두 자동으로 처리됩니다.
Firebase 동기화도 앱이 자동으로 수행하므로 별도 서버 없이 기기 간 데이터 공유가 가능합니다.
Section 2 · 배포 파이프라인
코드 변경 → 사용자 반영 흐름
단계별 흐름
1
코드 수정 & 커밋
src/App.jsx 수정 → git add → git commit
2
GitHub에 push

3

CDN 전파

웹 앱 접속 → 새 버전 자동 적용 (총 60~90초)
배포 확인 방법

VITE_FIREBASE_* 변수가 Vercel에 등록되지 않으면 빌드는 성공해도 Firebase 연결이 안 됩니다.
Section 2 · 데이터 흐름
데이터 동기화 & 환경변수 흐름
기기 간 동기화 흐름
[모바일]          [Firebase]          [PC]

지출 입력
↓ (5초 후 자동 push)
─────────────────────►
                      데이터 저장
                      ◄─────────────────
                      앱 시작 시 auto pull
                      ─────────────────►
                                        최신 데이터 표시

환경변수 주입 경로
Firebase Console → SDK 설정값 복사
          │
          ├──► .env.local         (개발 환경)
          │    ← gitignore, 로컬 전용
          │    → npm run dev에서 번들에 주입
          │
          └──► Vercel 환경변수    (운영 환경)
               → npm run build에서 번들에 주입
               → dist/ 파일에 포함되어 배포
⚠️ .env.local은 git에 커밋 금지

Firestore 보안 규칙
// Firestore → 규칙 탭에 입력 후 게시
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /sync/{syncKey} {
      allow read, write: if true;
    }
  }
}
